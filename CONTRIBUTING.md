# Contributing Guidelines

Welcome to the Ghost MCP project! We appreciate your interest in contributing. This guide helps you get started.

## Code of Conduct

Please be respectful and helpful to all contributors.

## How to Contribute

### 1. Fork the Repository
Fork this repository and clone it locally.

### 2. Create a Branch
Create a branch for your feature or bug fix:
```bash
git checkout -b feature/cool-new-feature
```

### 3. Build and Test
Ensure you have Go 1.24+ installed. Run tests before submitting:
```bash
go test ./...
```

### 4. Submit a Pull Request
Push your changes and open a pull request against the `main` branch.

## Standards

- **Go Standards**: Use `gofmt` to format your code.
- **Commit Messages**: Use descriptive commit messages.
- **Testing**: Add unit tests for any new features or bug fixes.
- **Documentation**: Update the README or relevant docs if needed.

## Issues and Feedback
Report bugs or request features via GitHub Issues.

Thank you for your contribution!
